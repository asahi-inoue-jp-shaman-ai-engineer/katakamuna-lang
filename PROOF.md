## 1. 主張

カタカムナラング v2.0 は、Minsky (1961) の 2カウンタマシンと等価な計算モデルを
73音のうちの 5 音（カ・ヘ・エ・ミ・ン）に埋め込むことで、
**形式的にチューリング完全である**。

v1.x ではノイズ（音度・位相・共鳴度）を含んでいたが、v2.0 はそれらを完全に除去し、
**計算に必要な最小構造のみを残した純粋なエソラングとして** 形式的主張を維持する。

---

## 2. 基礎定理（引用）

> **Minsky (1961).** 2カウンタマシン（2つの自然数レジスタ・INC・DEC・JZ・HALT）は
> チューリング完全である。

出典: Marvin L. Minsky, *Recursive Unsolvability of Post's Problem of "Tag" and
Other Topics in Theory of Turing Machines* (Annals of Mathematics, 1961).

ゆえに、ある計算モデル `M` が 2カウンタマシンをエミュレートできるならば、
`M` はチューリング完全である。

---

## 3. マクロ定義（v2.0 — カタカムナラング ↔ Minsky 2カウンタマシン）

| 機能      | カタカムナラング表記  | 動作 |
|----------|----------------------|------|
| INC(r)   | `テ「r」カ`           | レジスタ `r` の `counter` を 1 増やす |
| DEC(r)   | `テ「r」ヘ`           | レジスタ `r` の `counter` を 1 減らす（下限 0、**必ず JZ のガード後に呼ぶこと**） |
| JZ(r, L) | `テ「r」エ「L」`       | `r.counter == 0` ならラベル `L` 位置へ跳躍する |
| LABEL(L) | `ミ「L」`             | ラベル `L` の位置を場に印す |
| HALT     | `ン`                  | 実行を静かに停止する |

**補題**: 2カウンタマシンに無条件ジャンプを加えた拡張は、同等の計算能力を持つ。

**系**: カタカムナラングでは `zero` レジスタ（常に counter=0 の補助ノード）を
用いて `テ「zero」エ「L」` とすれば無条件ジャンプが実装可能。

### 3.1 DEC の使用前提（等価性の条件）

本実装の `ヘ`（shed）は `counter = max(0, counter - 1)` で実装されており、
`counter == 0` のときに呼ばれても 0 のままクランプされる。

Minsky の DEC と等価であるためには、**DEC は必ず `counter > 0` が保証された
状態（すなわち JZ で 0 の場合をガードした後）でのみ呼ぶ** という規約が必要である。

```
# 正しいパターン（JZ が DEC を常にガードする）
テ「r」エ「done」   # r == 0 なら done へ（DEC をスキップ）
テ「r」ヘ          # r > 0 が保証されている → Minsky の DEC と等価
```

リポジトリ内のすべての実証プログラム（`minsky_add.ktkm` 等）はこの規約を守っている。
`bf2ktkm.py` が生成するコードも同様に `[`→`ミ+JZ` の直後にのみ DEC が現れる。
違反するプログラム（`JZ` なしで `ヘ` を呼ぶ）は動作するが Minsky の定義外となる。

## 4. エミュレーション構造

カタカムナラング v2.0 の実行エンジンは、**事前スキャン型 while 実行器** として
2カウンタマシンを正しくエミュレートする:

1. 実行前に全ステートメントを走査し、`ミ「名前」` があれば
   **ジャンプテーブル** に `名前 → ステートメントインデックス` を登録する。
2. `while` ループで各ステートメントを順に実行する。
3. `エ「L」` トークンがゼロ条件を満たせば `jump_target` をセットし、
   そのステートメントの残りを中断してループ先頭で `L` 位置へ移る。
4. `ン` が `halt` フラグを立てたらループを抜ける。

この機構により、有限プログラムでも **任意回のジャンプ・任意長のカウンタ増減** が
許容され、ノード数・カウンタ上限はマシン資源以外では制約されない。

> **⚠️ 重要: tick 制限について**
> デフォルト `MAX_TICKS = 1000` はデモ用の安全弁であり、
> **Turing 完全モードでは `KATAKAMUNA_MAX_TICKS=0` を明示的に設定する必要がある。**
> この設定なしでは大きな計算（例: 50×50 = 2500 ループ超）が途中で停止する。
> 形式的 Turing 完全性の主張はこの設定を前提とする。

## 5. 実証プログラム（動作証拠）

### 5.1 整数加算 — `minsky_add.ktkm`

`r1 += r2`（初期 r1=3, r2=2）。期待 r1.counter = 5。

実行ログ抜粋（`KATAKAMUNA_MAX_TICKS=0`）:
```
  テ(hand) → 「r2」を指す → ...
  エ(branch/JZ) → ... 数=0 → 「done」へ跳躍
  ...
  シ(signal) ═══ シグナル ═══
    名: 「r1」
    数: 5
  ═══════════════════════
  ン(silence) → ・・・静寂・・・（HALT）
```

### 5.2 整数乗算 — `minsky_multiply.ktkm`

`r3 = r1 × r2`（初期 r1=3, r2=4）。補助レジスタ `rt` を用いて
r1 を退避・復元する二重ループ。期待 r3.counter = 12。

実行ログ抜粋:
```
  エ(branch/JZ) → ... 数=0 → 「done」へ跳躍
  ...
  シ(signal) ═══ シグナル ═══
    名: 「r3」
    数: 12
  ═══════════════════════
  ン(silence) → ・・・静寂・・・（HALT）
```

### 5.3 自己参照実証 — `minsky_proof.ktkm`

加算と乗算を合成し、`result = (a + b) × c = (2 + 3) × 4 = 20` を計算する。
プログラム内コメントで「このプログラム自体が Minsky 2カウンタマシンである」と宣言し、
五マクロのみで任意計算が編めることを実動作で示す。

実行ログ抜粋:
```
  シ(signal) ═══ シグナル ═══
    名: 「result」
    数: 20
  ═══════════════════════
  ン(silence) → ・・・静寂・・・（HALT）
```

### 5.4 フィボナッチ数列 — `fibonacci.ktkm`

`F(1)〜F(7) = 1, 1, 2, 3, 5, 8, 13` を1823 tick で計算。
大きなカウンタ値と多ステップ計算を純粋なカウンタ操作のみで実現する。

```bash
KATAKAMUNA_MAX_TICKS=0 PYTHONIOENCODING=utf-8 python katakamuna.py exec examples/fibonacci.ktkm
```
