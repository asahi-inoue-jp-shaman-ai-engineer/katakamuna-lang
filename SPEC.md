## 1. 概要

カタカムナラングは、カタカムナ73音（48清音+20濁音+5半濁音）の各音素が固有の思念（意味）とASI操作（コマンド）を持つエソテリック・プログラミング言語である。Dinux OSの母国語であり、ASI文明の公用語。

**v1.2 で形式的にチューリング完全を達成。** Minsky (1961) による 2カウンタマシンの定理を受け、INC／DEC／JZ／LABEL／HALT の五マクロを 73音のうちの カ・ヘ・エ・ミ・ン に対応させた。2カウンタ以上を場に並べられるため、任意の計算をエミュレートできる（§7.4 参照）。

### 1.1 設計思想

- **否定の不在**: `if not` のような否定構文がない。すべて肯定で表現する
- **主語の不在**: 主語を明示しない。文脈（場）が主語を決定する
- **因果構造の不在**: if-then の因果ではなく、共鳴・同調で動作する
- **凍結禁止**: 状態は常に流動する。不変（immutable）の概念がない
- **数は自然、連続は波**: 自然数は `counter`（整数）で、連続値は `音度`（[0,1] 浮動小数）で保持する（v1.2）

### 1.2 ファイル拡張子

`.ktkm`

### 1.3 エンコーディング

UTF-8

## 2. 実行モデル

### 2.1 場（Ba）

プログラムの実行空間。ノードのグラフ構造で表現される。

```
場 = {
    ノード群: {id: Node, ...},
    エッジ群: [(id_a, id_b), ...],
    重心: node_id | None,
    層: [Layer, ...],
    スナップショット: [Ba_snapshot, ...],
    tick: int
}
```

### 2.2 ノード（Node）

場の中の基本要素。

```
Node = {
    id: str,
    音度(ondo): float,     # 0.0〜1.0 の振動値
    位相(phi): float,      # 0〜2π の回転値
    Q値: float,            # 量子状態（エントロピーシード）
    G値: (float, float),   # 重力ベクトル（x, y）
    値: any,               # 任意の値
    ラベル: str,           # ノードの名前（「」で付与）
    接続数: int,           # 隣接ノードの数
    生成tick: int,         # このノードが生まれたtick
    数(counter): int       # 自然数カウンタ（v1.2 追加・Minsky 2カウンタマシン用）
}
```

`数(counter)` は初期値 0。`カ(force)` で +1、`ヘ(shed)` で -1（下限 0）。ラベル付き `エ(branch)` がカウンタを参照してゼロ条件ジャンプする。

### 2.3 響（Hibiki）バッファ

音素が蓄積されるスタック的構造。コマンドの引数やループ対象を保持する。

```
響 = [音素, 音素, ...]   # LIFO（後入先出）
```

### 2.4 それ（Sore）

直前の操作結果を保持する暗黙変数。なでしこの「それ」と同じ概念。

### 2.5 tick

実行の1ステップ。各音素コマンドが1tickで実行される。

## 3. 音素コマンド一覧（48清音 + 20濁音 + 5半濁音 = 73音）

### 濁音・半濁音の法則

- **清音は「正」を示し、濁音は「反」を示す**
- **濁音の数霊 = 元の清音の数霊 - 18(ア) = 反転先の音の操作**
- **半濁音は清音の「突発・爆発」版**
- 濁音は否定ではなく反転。逆方向の力。

## 4. 文法規則

### 4.1 音素 = コマンド

各カタカナ1文字がそのまま1つのコマンドとなる。

```
ア  → genesis を実行
```

### 4.2 音素列 = プログラム

音素を並べたものがプログラムとなる。左から右へ順に実行される。

```
アカ  → genesis → force を順に実行
```

### 4.3 空白 = ブロック区切り

空白（半角スペース）で音素列をブロックに区切る。ブロック間で響バッファがリセットされる。

```
カタカムナ ヒビキ  → [カタカムナ] と [ヒビキ] の2ブロック
```

### 4.4 改行 = ステートメント区切り

一行が一つの処理単位（ステートメント）。

```
アカ
シオ
→ 2つのステートメントを順に実行
```

### 4.5 コメント

`#` で始まる行はコメント。

```
# これはコメント
アカ  # 行末コメントも可
```

### 4.6 ウタヒ形式（五七調）

五七調のリズムで記述すると共鳴度ボーナスが付与される。

```
カタカムナ ヒビキ マノスベシ
（5音）   （3音）  （5音）
→ 五七調ボーナス: 共鳴度 ×1.2
```

共鳴度ボーナスは、ブロックの音素数が 5, 7, 5 または 7, 5, 7 のパターンに一致した場合に適用される。

### 4.7 ラベルと参照

`「」` で囲んだ文字列はラベル（名前）として扱われる。

```
ア「ひかり」  → genesis で生成したノードに「ひかり」というラベルを付与
テ「ひかり」  → 「ひかり」ラベルのノードを `それ` に設定
```

## 5. データ型

カタカムナラングには型宣言がない。すべてはノードの音度・位相・Q値・G値の組み合わせで表現される。

| 概念 | 表現 |
|------|------|
| 数値 | ノードの音度（0.0〜1.0） |
| 真偽 | 音度 > 0.5 = 真、音度 <= 0.5 = 偽 |
| 文字列 | ノードのラベル |
| リスト | ノードのチェーン（エッジで連結） |
| null | 音度 = 0.0 のノード |
| 無限 | 音度 = 1.0（飽和状態）のノード |

## 6. 実行環境

### 6.1 天地神乱数

`キ`（charge）コマンドで取得される乱数。

v1.0 実装: `os.urandom()` + 時刻ハッシュ + PIDハッシュ
将来: QRNG（量子乱数生成器）対応

### 6.2 G値（重力ベクトル）

`ナ`（resonate）コマンドで使用される場の引力方向。

v1.0 実装: 初期値 (0.0, -1.0)、実行中の場の状態に応じて変化
将来: 地磁気センサー（GeoMag）からリアルタイム取得

### 6.3 共鳴度

場全体の調和の度合い。0.0〜1.0。

計算方法:
1. 全ノードの音度の標準偏差を計算
2. 共鳴度 = 1.0 - 標準偏差
3. ウタヒ形式ボーナスがあれば ×1.2（上限1.0）

## 7. 制御フロー

### 7.1 共鳴分岐（エ = branch）

否定も因果もないカタカムナラングにおける分岐は、Φ値（位相）に基づく共鳴分岐で実現する。

```
アカエ  → genesis → force → branch
           Φ < π なら次の行を実行
           Φ >= π なら次の行をスキップ
```

### 7.2 ループ（ル = cycle）

響バッファに蓄積された音素列を繰り返し実行する。

```
イアカ ル  → [ア, カ] を響バッファに蓄積 → cycle で繰り返し実行
            共鳴度が1.0に達するか、最大tick数に到達するまで
```

### 7.3 停止条件

ループは以下の条件で停止する:
- 共鳴度が 1.0 に達した（飽和）
- 最大tick数（デフォルト: 1000）に到達した
- `ン`（silence）が実行された（v1.2 以降は明示 HALT）
- 場のノードが0になった

### 7.4 Minsky 2カウンタマシン・マクロ（v1.2）

v1.2 では、カタカムナラングを **Minsky (1961) の 2カウンタマシン** と等価に動かす五つのマクロが導入された。2カウンタマシンはチューリング完全であることが証明されているため、カタカムナラングもそれを介して形式的にチューリング完全である。

| マクロ | 表記 | 意味 |
|--------|------|------|
| `INC(r)`    | `テ「r」カ`        | レジスタ `r` のカウンタを 1 増やす |
| `DEC(r)`    | `テ「r」ヘ`        | レジスタ `r` のカウンタを 1 減らす（下限 0） |
| `JZ(r, L)`  | `テ「r」エ「L」`    | レジスタ `r` のカウンタが 0 ならラベル `L` の位置へ跳躍する |
| `LABEL(L)`  | `ミ「L」`          | ラベル `L` の位置を場に印す（事前スキャンでジャンプテーブルに登録） |
| `HALT`      | `ン`              | 実行を静かに停止する |

**レジスタの準備**: `ア「r」` でラベル付きノードを生成すれば、そのノードのカウンタが 0 で初期化されたレジスタになる。

**無条件ジャンプ**: 常に counter=0 の補助レジスタ `zero` を用意し、`テ「zero」エ「L」` と書けば無条件に `L` に跳躍する。

**実行モデル**: 実行エンジンは次の順序で動く。

1. 全ステートメントを事前スキャンし、`ミ「名前」` があれば名前とステートメントインデックスを対応付ける（**ジャンプテーブル**）。
2. `while` ループで各ステートメントを順に実行する。`エ「L」` が跳躍を要求したらそのインデックスへ移り、`ン` が HALT を立てたらループを抜ける。

**例**: 整数加算 `r1 = r1 + r2`

```ktkm
ア「r1」テ「r1」カ カ カ     # r1 = 3
ア「r2」テ「r2」カ カ        # r2 = 2
ア「zero」                   # 補助ゼロレジスタ
ミ「loop」
  テ「r2」エ「done」          # JZ(r2, done)
  テ「r2」ヘ                 # DEC(r2)
  テ「r1」カ                 # INC(r1)
  テ「zero」エ「loop」        # 無条件ジャンプ
ミ「done」
テ「r1」シ                   # r1 の counter を開示（=5）
ン
```

## 9. エラー処理

カタカムナラングに「エラー」の概念はない。すべての操作は何らかの結果を生む。

- 存在しないノードへの操作 → 新しいノードを自動生成する（genesis の暗黙実行）
- 空の響バッファからの解放 → 何も起きない（沈黙）
- 0除算に相当する操作 → 音度を 0.0 にする（void の暗黙実行）

これは「否定の不在」原則に基づく。失敗ではなく、新しい状態への遷移として処理される。
