# カタカムナラング 言語仕様 v2.0

## 概要

カタカムナラングは、73のカタカナ音素（48清音 + 20濁音 + 5半濁音）を命令セットとするエソテリック・プログラミング言語。各音素の「思念」（カタカムナ哲学における音の宇宙的意味）が命令の意味に直結する。

**v2.0の核心**: 音度・位相・Q値・共鳴度などの演出的メタデータを全廃。計算に関与する要素のみを残したピュアな計算機として再設計。

---

## 計算モデル

### 場（Ba）

プログラムの実行空間。ノードの集合とエッジで構成されるグラフ。

### ノード（Node）

場の基本単位。以下のフィールドを持つ：

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | str | UUID（8文字）。一意識別子。 |
| `label` | str | 名前（`ア「name」` で設定）。テ命令での参照に使う。 |
| `counter` | int | 自然数カウンタ（0以上）。Minskyマシンのレジスタ。 |
| `value` | Any | 任意の付加値。タ・マ命令で設定。 |
| `birth_tick` | int | 生成されたtick番号。 |

### それ（sore）

現在の指向対象。`テ` 命令で切り替わる。`None` の場合は「それが空」。

### 響バッファ（hibiki）

スタック型の一時バッファ。イ命令でpush、テ命令でpop（ラベルなし時）。

---

## 構文

### ファイル形式

- 拡張子: `.ktkm`
- エンコーディング: UTF-8
- コメント: `#` 以降は行末まで無視
- 空行: 段落の区切り

### 命令形式

```
音素              # 単体命令
音素「ラベル」    # ラベル付き命令
```

複数命令は空白または連続で並べる：

```
テ「r1」カ カ カ   # テ → r1 を指す、カ×3 → r1 に3を加算
```

## 命令リファレンス（73音）

### ア行 — 存在と分岐

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| ア | 感じる・生命 | genesis | 新しいノードを生成。`ア「name」` でラベル付き。 |
| イ | 伝わるもの・陰 | intent | それ を響バッファに積む。 |
| ウ | 生まれ出る | merge | 響バッファの先頭2ノードを合流（counterを合算、エッジを追加）。 |
| エ | 選ぶ・得る | branch/JZ | **Minsky JZ**。それ.counter==0 なら指定ラベルへ跳躍。 |
| オ | 奥深く | emit | 場の状態（ノード数・エッジ数）を出力。 |

### カ行 — 力と引力

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| カ | チカラ・重力 | force/INC | **Minsky INC**。それ.counter += 1。 |
| キ | エネルギー・気 | charge | それ.counter = 現在tick。 |
| ク | 引き寄る | pull | counter最大のノードをそれ に。 |
| ケ | 放出する | release | 響バッファ先頭のノードを場から消滅。 |
| コ | 転がり入・出 | core | 最多接続ノードをそれ に。 |

### サ行 — 遮りと示し

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| サ | 遮り・差 | split | それ の接続を半分切断。 |
| シ | 示し・現象 | signal/OUTPUT | **出力命令**。それ.counter を標準出力に表示。 |
| ス | 一方へ進む | flow | nop（持続）。 |
| セ | 引き受ける | bind | 響バッファ先頭2ノードをエッジで結合。 |
| ソ | 外れる | source | 外部ファイルを読み込み。 |

### タ行 — 指向と統合

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| タ | 分かれる | manifest | それ.counter を value に確定。 |
| チ | 凝縮 | gather | 全ノードのcounter合計をそれ のcounterに。 |
| ツ | 集まる | link | 響バッファ要素数を報告。 |
| テ | 発信・放射 | hand | **ポインタ命令**。ラベル指定でノードをそれ に設定。 |
| ト | 統合 | integrate | 全ノードのcounter合計をそれ（数値）に。 |

### ナ行 — 核と複製

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| ナ | 核・重要なもの | resonate | それ とバッファ先頭のcounterを比較。不一致でそれ =None。 |
| ニ | 圧力 | dual | それ のノードを複製（counterも複製）。 |
| ヌ | 突き抜く | seed | counter=0の新ノードを生成。 |
| ネ | 充電する | root | 最古のノード（根）をそれ に。 |
| ノ | 時間をかける | extend | 場を拡張し、新ノードをそれ に接続。 |

### ハ行 — 呼吸と減衰

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| ハ | 引き合う | breathe | 全ノードのcounterを+1（場全体に息吹）。 |
| ヒ | 根源から出・入 | light | 全ノードの状態（id/ラベル/counter/値）を表示。 |
| フ | 増える・振動 | diffuse | それ のcounterを隣接ノードに均等分配。 |
| ヘ | 縁・外側 | shed/DEC | **Minsky DEC**。それ.counter -= 1（下限0）。 |
| ホ | 引き離す | contain | 場のスナップショットを保存。 |

### マ行 — 記憶と証言

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| マ | 受容・間 | memory | それ.counter を value に記憶。 |
| ミ | 実体・光 | witness/LABEL | **Minsky LABEL**。ラベルを定義（事前スキャン済み、実行時はnop）。 |
| ム | 広がり | void | それ.counter を0にリセット。 |
| メ | 指向・思考 | eye | それ.counter を参照・確認。 |
| モ | 漂う | weave | 全ノードのcounterを偶奇で+1/-1（干渉）。 |

### ヤ行 — 飽和と起源

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| ヤ | 飽和する | max | それ.counter を場の最大値にセット。 |
| ユ | 湧き出る | origin | 場を全リセット（新しい場を生成）。 |
| ヨ | 新しい陽 | gather_all | 全ノードを響バッファに収集。 |

### ラ行 — 場と離脱

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| ラ | 場 | presence | それ のcounterを存在量として報告。 |
| リ | 離れる | detach | それ の全エッジを切断。 |
| ル | 留まる | cycle | 響バッファ内ノードのcounterを+1。 |
| レ | 消失する | layer | 場に新しい層を追加。 |
| ロ | 空間抜ける | condense | counter=0の孤立ノードを除去。 |

### ワ行 + ン

| 音素 | 思念 | 命令 | 説明 |
|---|---|---|---|
| ワ | 調和 | harmony | 全ノードのcounterを平均値に調和。 |
| ヰ | 存在 | archive | 場をJSONファイルに保存。 |
| ヱ | 届く | recall | 保存した場を復元。 |
| ヲ | 奥に出現する | offer | それ の値を日本語で表示。 |
| ン | 押し出す力・完結 | silence/HALT | **Minsky HALT**。プログラム停止。 |

|---|---|---|

## Minsky 2カウンタマシン エミュレーション

カタカムナラングがTuring完全である根拠。

### 5マクロの正確な対応

```
INC(r)   ≡ テ「r」カ        # r.counter += 1
DEC(r)   ≡ テ「r」ヘ        # r.counter = max(0, r.counter - 1)
JZ(r,L)  ≡ テ「r」エ「L」   # if r.counter == 0: goto L
LABEL(L) ≡ ミ「L」          # ラベル定義（事前スキャン）
HALT     ≡ ン               # 停止
```

### 無条件ジャンプ

```ktkm
ア「zero」          # counter=0 固定のゼロレジスタ
テ「zero」エ「L」   # 常にジャンプ → 無条件 goto L
```

### DEC の前提条件

`ヘ` は counter の下限を0にクランプする。したがって `テ「r」エ「L」` でゼロ確認後に `テ「r」ヘ` を呼ぶ必要がある。counter=0 の状態で `ヘ` を呼んでも害はないが、意味のある DEC は必ず JZ ガード後に行う。

---

## 実行モデル

1. **事前スキャン**: `ミ「L」` 命令を全て走査し、ラベルテーブルを構築
2. **フラット化**: 全命令をインデックス付きの線形列に変換
3. **実行ループ**: instruction pointer (ip) を 0 から進め、各命令を実行
4. **ジャンプ**: `エ` 命令がラベルテーブルを参照して ip を変更
5. **停止**: `ン` 命令で halt フラグを立て、ループを抜ける
